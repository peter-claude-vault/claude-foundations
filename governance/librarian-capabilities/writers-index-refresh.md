## Purpose

Sweep all writer-reference markdown files in `$VAULT_ROOT/Vault Writers/` and regenerate the canonical `$VAULT_ROOT/Vault Writers/_index.md` catalog table. The capability reads each writer-reference's frontmatter (validated against `governance/file-type-contracts/vault-writer.md.json`) and emits a unified table summarizing all registered writers: Name / Kind / Status / Last run / Destinations summary. The `_index.md` body also carries a short folder-context paragraph (preserved across regeneration via sentinel-marker pattern) and a `## See also` H2 pointer-link to the sibling `_overlap-matrix.md` (regenerated by `writers-overlap-refresh` capability T-12.4).

The capability is the canonical writer for `Vault Writers/_index.md`. The file declares `writers_allowed: ["librarian"]` in its frontmatter per Session 5 L-68, enforcing the librarian-exclusive write posture. Hook branch #4 enforcement (per SP13 Session 6 packet) blocks non-librarian writes to this path.

## Output Contract

**Files written:**
- `$VAULT_ROOT/Vault Writers/_index.md` â€” regenerated catalog file (`type: index`; `generated_by: librarian writers-index-refresh`; `writers_allowed: ["librarian"]`; `updated: <ISO>`; standard tags). **Schema each is gated by:**
- Every writer-reference markdown read validates frontmatter against `governance/file-type-contracts/vault-writer.md.json` (required fields + conditional-required by `writer_kind` + enums) BEFORE row composition.
- `_index.md` written frontmatter validates against `governance/frontmatter-rules.json#types.index` (`type: index`; `parent_folder: "Vault Writers"`; `writers_allowed`; `tags`; `updated`).
- `_index.md` body table conforms to the column shape locked in Session 5 L-67 (Name / Kind / Status / Last run / Destinations summary).
- NDJSON output validates against `librarian-finding-schema.json`.
- Manifest subtree mirror validates against `librarian-manifest-schema.json` `drift_findings.writers_index_refresh` (open `additionalProperties: true` object).

**Pre-write validation steps:**
- Read `governance/file-type-contracts/vault-writer.md.json` (writer-reference contract); validate the contract itself against `schemas/file-type-contract-schema.json` (when present in foundation-repo; Draft-07).
- Read `governance/vault-writers-rules.json` operational config (for `output_type_enum` / `posture_enum` reference shapes embedded in `destinations[]` validation).
- Read `$VAULT_ROOT/Vault Writers/*.md` writer-reference files (exclude `_index.md` + `_overlap-matrix.md`).
- For each writer-reference: validate frontmatter against the contract. On failure: emit `writer-frontmatter-malformed` finding; still render row IF `writer_name` is parseable; otherwise skip the file.
- Required-field check: `type`, `writer_name`, `writer_kind`, `writer_skill`, `destinations`, `status`, `created`, `updated`, `tags`. Missing required â†’ `writer-missing-required-field` finding.
- Conditional-required check (per `writer_kind`):
  - `writer_kind: connector` requires `writer_subtype` + `source` + `authentication`
  - `writer_kind: agentic-flow` requires `source`
  - `writer_kind: auto-research` requires `source` + `schedule`
  - `writer_kind: scheduled-skill` requires `schedule`
  Missing conditional â†’ `writer-kind-violation` finding.

**Failure mode:**
- `block and log` on schema-validation failure of source contracts. The capability does not partially regenerate when its inputs are malformed.
- `block and log` on `_index.md` write that would fail post-write frontmatter validation against `#types.index`.
- `block and log` if the sentinel-marker pair (`<!-- writers-index:start -->` / `<!-- writers-index:end -->`) cannot be located in the existing `_index.md` AND a fresh file cannot be initialized.
- Never `write and hope`. Atomic temp+rename for the regeneration; operator narrative outside sentinels preserved across the swap.

## Behavior â€” full regeneration

1. **Read contract.** Load `governance/file-type-contracts/vault-writer.md.json` + `governance/vault-writers-rules.json` operational config.
2. **Enumerate writer-references.** Walk `$VAULT_ROOT/Vault Writers/*.md`; exclude `_index.md` + `_overlap-matrix.md` + any file with leading underscore.
3. **Per-writer validation.** For each file: parse YAML frontmatter; validate against contract (required + conditional + enums); emit findings on failure but continue rendering.
4. **Compose rows.** Per Session 5 L-67 shape:
   - `Name` â€” `writer_name` field
   - `Kind` â€” `writer_kind` field
   - `Status` â€” `status` field (`active / paused / retired`)
   - `Last run` â€” `last_run` field (librarian-maintained; `â€”` if absent)
   - `Destinations summary` â€” count + first-destination-path summary from `destinations[]` array
5. **Preserve operator narrative.** Read existing `_index.md`; capture content outside the `<!-- writers-index:start -->` / `<!-- writers-index:end -->` sentinel pair (folder-context paragraph + `## See also` H2 pointer-link to `_overlap-matrix.md`).
6. **Render new file.** Compose:
   - Frontmatter (`type: index`, `parent_folder: "Vault Writers"`, `generated_by: librarian writers-index-refresh`, `writers_allowed: ["librarian"]`, `tags`, `updated: <today>`)
   - H1 (`# Vault Writers`)
   - Preserved folder-context paragraph
   - `## Writers` H2
   - Sentinel-start marker
   - Markdown table with column headers + separator + rows (sorted by `Name` ascending)
   - Sentinel-end marker
   - `## See also` H2 with pointer-link to `[[_overlap-matrix]]` (NOT embedded matrix per L-68)
7. **Atomic write.** Write tempfile; validate parseability + frontmatter; atomic rename.
8. **Emit `writers-index-regenerated` event finding** with counts.

## Self-healing boundary declaration (R-34)

This capability operates under the R-34 mechanical-mutation boundary:

- **In bounds (auto-corrected):** Table region inside sentinel markers â€” rows added / removed / re-sorted; cells populated from writer-reference frontmatter; frontmatter `updated:` bumped.
- **Out of bounds (flagged, never overwritten):** Folder-context paragraph (operator-tunable); `## See also` H2 body (operator may add adjacent pointer-links); writer-reference frontmatter (capability is read-only against writer-references); `_overlap-matrix.md` (separate capability writes that file).
