# tests/grep-audit-fixtures/

Deliberate identity-leak fixtures for the four-layer grep-audit. Each fixture is a known-bad sample that exactly one layer should catch.

The grep-audit walks four layers in sequence — raw ASCII, NFKC-normalized Unicode, base64-decoded, and git-history (committed-then-removed). Each fixture targets one layer specifically.

## When this fires

The audit excludes `tests/grep-audit-fixtures/` from its default scan path. The fixtures only fire when a test harness points the audit at this directory explicitly. That means a normal audit run against the repo never trips on these files; they're a positive-control set for the audit harness.

## Inventory

| Fixture | Layer | Hit string | How |
|---|---|---|---|
| `fixture-layer1-raw.txt` | 1 | a literal ASCII identity token | exact-string grep |
| `fixture-layer2-nfkc.txt` | 2 | a Unicode bold variant of the same token | NFKC normalization unwraps the styling |
| `fixture-layer3-base64.txt` | 3 | the same token, base64-encoded | base64 decode then grep |
| `fixture-layer4-history` | 4 | the same token, committed and then removed | `git log -p` reads the removed content |

Each layer adds a transformation pass that catches a class of leak the prior layers miss. Layer 1 catches typos. Layer 2 catches "stylized in Markdown" copy-paste. Layer 3 catches embedded blobs. Layer 4 catches "I removed it but didn't rewrite history."

## Generation

Layer 2 (Unicode in git is brittle across editors) and Layer 3 (base64 blobs are opaque to review) are generated by `tests/grep-audit-fixtures/generate.sh` rather than committed as-is. Run that script before the harness if the layer-2 or layer-3 file is missing or stale.

Layers 1 and 4 are committed verbatim — they're either trivial (raw text) or load-bearing on git history (the layer-4 fixture is a directory whose `.git` is part of the test surface).

## Adding a new fixture

If you find a leak class the four current layers miss, add:

- A new fixture file under this directory.
- A new layer to the audit harness that catches it.
- A row in the inventory table above.

The "catches it" property is the test: the new layer must hit the new fixture, and no prior layer must hit it (otherwise the new layer is redundant). The harness asserts both.
