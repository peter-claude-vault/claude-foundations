# grep-audit fixtures

Each fixture is a deliberate leak that exactly one layer should catch. The
fixtures live under `tests/grep-audit-fixtures/` which is excluded from the
default audit path — they only fire when a test harness points the audit at
this directory explicitly.

| Fixture | Layer | Hit string | How |
|---|---|---|---|
| fixture-layer1-raw.txt | 1 | `petertiktinsky` | literal ASCII |
| fixture-layer2-nfkc.txt | 2 | Unicode bold `𝐏𝐞𝐭𝐞𝐫` | NFKC normalization catches it |
| fixture-layer3-base64.txt | 3 | b64-encoded `petertiktinsky` | base64 decode catches it |
| fixture-layer4-history | 4 | committed then removed | git-log diff catches it |

Fixtures are generated by `tests/grep-audit-fixtures/generate.sh` rather than
committed as-is for Layer 2 (Unicode in git is brittle across editors) and
Layer 3 (base64 blobs are opaque to review).
